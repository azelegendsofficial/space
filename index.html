Definisi Server

"Penyedia Layanan"
Model Komunikasi: Klien-Server

Server adalah sebuah **sistem komputer** (baik perangkat keras, perangkat lunak, atau keduanya) yang didesain untuk **memproses permintaan** dan **mengirimkan data** kepada komputer lain, yang disebut **klien**, melalui jaringan.

Bayangkan server sebagai **pelayan digital** yang menyimpan semua sumber daya yang diperlukan agar website, aplikasi, atau layanan email dapat berfungsi.

Fungsi Utama Server:

  • Menyimpan Data: Menyimpan *file* website, data aplikasi, basis data, dan informasi pengguna.
  • Memproses Permintaan: Menerima permintaan dari klien dan menjalankan logika yang diperlukan untuk menghasilkan respons.
  • Mengirimkan Sumber Daya: Mengirimkan kembali data yang diminta (seperti halaman HTML, gambar, atau video) kepada klien.

Cara Kerja Server

"Siklus Klien-Server"

Mekanisme kerja server mengikuti model **Klien-Server** dalam siklus langkah demi langkah:

1. Klien Membuat Permintaan (Request)

Ketika Anda mengetik alamat website (**URL**) di *browser* (klien), perangkat Anda mengirimkan paket data‚Äîsebuah **permintaan**‚Äîmelalui internet menuju alamat IP spesifik server tersebut.

2. Server Menerima & Memproses

Server menerima permintaan. Jika itu adalah permintaan untuk halaman *web*, server: a) Mencari *file* yang diminta. b) Berinteraksi dengan basis data (jika perlu). c) Menjalankan *script* aplikasi (PHP, Python, dll.) untuk menyusun konten dinamis.

3. Server Mengirimkan Respon (Response)

Setelah pemrosesan selesai, server mengirimkan kembali data yang telah disusun (misalnya, kode HTML lengkap) sebagai **respons** kepada klien, biasanya menggunakan protokol **HTTP**.

4. Klien Menampilkan Data

*Browser* (klien) menerima respons dan menerjemahkan kode-kode tersebut untuk **menampilkan halaman *web*** yang dapat Anda lihat dan interaksikan.
